Git Workflow:

1.Start your work from the develop branch.
2.Create a new feature branch for your changes.
3.After finishing the work, commit with a clear message.
4.Push your branch and open a pull request to the develop branch

Pull Request Guidelines:

1.Use a clear title and a simple description.
2.Mention issue numbers if your PR fixes something.
3.Make sure your code works before creating the PR.
4.If any changes are requested, update your PR.

Code Standards:

1.Write clean and readable code.
2.Avoid unwanted files and unused comments.
3.Use proper naming for files, variables, and branches.
4.Commit messages should clearly say what you changed.
